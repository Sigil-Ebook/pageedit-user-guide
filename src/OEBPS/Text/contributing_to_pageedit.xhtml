Contributing to PageEdit

— Helping Development —

PageEdit is completely free and open-source software, and it is developed and supported entirely by volunteers who love books and want well-edited ebooks to be easily available. Your entirely optional contributions can help improve PageEdit and show your appreciation for making the software available. Feel free to just say hi on the PageEdit forum and let others know how useful PageEdit is to you.

Translating PageEdit

Translating PageEdit into other languages is highly encouraged. There is an active community of volunteers who translate PageEdit already, and new languages can always be added.

The best way to translate PageEdit is to use the web based translation system provided by Transifex. Translations done using Transifex are downloaded just before releasing a new version:

  • PageEdit translations @ Transifex

Alternatively you can use local Desktop tools for translation. PageEdit uses Qt’s Linguist tool for creating and editing translations. You can either download Qt and compile the tools manually or use the pre-built packages:

  • Pre-built Translation tools

Updating and creating new translation files using the Desktop tools is the same process:

  • Open a terminal and change to the src/PageEdit/Resource_Files/ts directory. This holds all the raw translation files.
  • Run lupdate ../../ * -ts out.ts to either generate a new translation file or to update an existing one with any new strings that have been added.
  • Use Linguist to write translations for the strings in a given ts file.

All translations files should have the form base_pageedit_lang.ts. Where lang is a two letter language code. For example the Polish translation will have the file name base_pageedit_pl.ts.

Assisting Users

If you are an experienced PageEdit user, HTML expert, or working your way there, you can help by answering questions or commenting on your experiences – primarily at the official PageEdit Forum at MobileRead. You can also check issues on the development website and add comments.

Coding

If you are a C++ programmer and have suggestions for improvements or bug fixes you are welcome to submit patches for consideration.
